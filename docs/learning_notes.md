# Week 1 Learning Notes

## Overview

This document summarizes the major concepts and learnings from Week 1 of the CareerPulse project.

---

## Key Learnings

### 1. Data Engineering Project Lifecycle

Understood the overall lifecycle of a Data Engineering project:

* Data Collection
* Data Ingestion
* Data Storage
* Data Transformation
* Data Visualization

---

### 2. Project Architecture Design

Learned how to design an end-to-end data pipeline.

Pipeline Flow:
Dataset + API
↓
Python Ingestion
↓
PostgreSQL Warehouse
↓
dbt Transformations
↓
Dashboard

---

### 3. Synthetic Dataset Creation

Learned how to generate a realistic synthetic dataset using Python.

Dataset contains:

* Job ID
* Job Title
* Company
* Location
* Salary
* Experience
* Skills
* Job Type
* Posted Date

---

### 4. Role-Based Data Generation

Learned how to create realistic data using role-based skill mapping.

Example:

* Data Engineer → Python, SQL, Airflow
* Data Analyst → SQL, Excel, Power BI

This improved dataset quality.

---

### 5. GitHub Project Structure

Learned how to organize project folders and documentation in GitHub.

Main folders:

* docs
* data
* ingestion

---

## Summary

Week 1 focused on project planning, architecture design, dataset creation, and repository setup.
